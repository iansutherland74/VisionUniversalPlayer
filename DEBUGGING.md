# VisionOS 26 Video Player - Debugging Checklist\n\nSystematic guide for diagnosing and fixing common issues.\n\n## Pre-Build Checklist\n\n### Project Setup\n- [ ] All source files copied to Xcode project\n- [ ] File memberships assigned to VisionUniversalPlayer target\n- [ ] Info.plist included in project\n- [ ] Bridging header path correct in Build Settings\n\n### Build Settings\n- [ ] Deployment Target: visionOS 2.0+\n- [ ] Supported Platforms: visionOS\n- [ ] Swift Language Version: 5.9+\n- [ ] `SWIFT_OBJC_BRIDGING_HEADER` set\n\n### Frameworks Linked\n- [ ] Metal (Build Phases → Link Binary)\n- [ ] MetalKit\n- [ ] VideoToolbox\n- [ ] AVFoundation\n- [ ] Combine\n\n### FFmpeg Setup\n- [ ] FFmpeg installed: `brew list ffmpeg` shows paths\n- [ ] HEADER_SEARCH_PATHS includes /opt/homebrew/include\n- [ ] LIBRARY_SEARCH_PATHS includes /opt/homebrew/lib\n- [ ] OTHER_LDFLAGS includes `-lavformat -lavcodec -lavutil -lpthread`\n\n## Build Errors\n\n### Error: \"Module 'VisionUniversalPlayer' has conflicting definitions\"\n\n**Symptoms**: Multiple C declarations, header conflicts\n\n**Fix**:\n1. Clean build folder: ⇧⌘K\n2. Delete DerivedData: `rm -rf ~/Library/Developer/Xcode/DerivedData/*`\n3. Close Xcode\n4. Open project again\n5. Build: ⌘B\n\n### Error: \"Linker command failed with exit code 1\"\n\n**Check FFmpeg:** \n```bash\n# Verify libraries exist\nls /opt/homebrew/lib/libav*.a\n\n# Should show:\n# libavcodec.a\n# libavformat.a\n# libavutil.a\n```\n\n**Fix if missing**:\n```bash\nbrew uninstall ffmpeg\nbrew install ffmpeg --HEAD  # Get latest\n```\n\n**Update Build Settings**:\n- `LIBRARY_SEARCH_PATHS = /opt/homebrew/lib`\n- `OTHER_LDFLAGS = -lavformat -lavcodec -lavutil -lpthread`\n\n### Error: \"Cannot find 'ffmpeg_open' in scope\"\n\n**Symptom**: Swift can't see C functions\n\n**Fix**:\n1. Check bridging header exists: `VisionUniversalPlayer-Bridging-Header.h`\n2. Verify it contains: `#include \"FFmpegBridge.h\"`\n3. In Build Settings, set:\n   ```\n   SWIFT_OBJC_BRIDGING_HEADER = VisionUniversalPlayer/VisionUniversalPlayer-Bridging-Header.h\n   ```\n4. Clean: ⇧⌘K\n\n### Error: \"FFmpegBridge.h: file not found\"\n\n**Fix**:\n1. Verify file location: `/Users/sutherland/vision ui/VisionUniversalPlayer/Engine/FFmpegBridge.h`\n2. File is added to Xcode project\n3. Update HEADER_SEARCH_PATHS:\n   ```\n   HEADER_SEARCH_PATHS = $(inherited) /opt/homebrew/include $(SRCROOT)/VisionUniversalPlayer/Engine\n   ```\n\n## Runtime Errors\n\n### App Crashes at Startup\n\n**Check console output** (Xcode → View → Debug Area → Show Console)\n\n**If error mentions \"Metal\"**:\n```\n[MTLDebugRenderCommandEncoder validateRenderPipelineState]\n```\n- Issue: Shader compilation failed\n- Fix: Check [Rendering/Shaders.metal](Rendering/Shaders.metal) for syntax errors\n\n**If error mentions \"VideoToolbox\"**:\n```\nVTDecompressionSessionCreate failed\n```\n- Possible: Codec not supported\n- Debug: Add print in `VideoDecoder.recreateDecompressionSession()`\n\n**If error mentions \"FFmpeg\"**:\n```\nffmpeg_open returned -4\n```\n- Issue: Cannot open URL\n- Debug: Check URL is reachable: `curl http://your-url.com/video.mp4`\n\n### Video Doesn't Play\n\n**Check PlayerViewModel logs**:\n1. Run app in debugger\n2. Open console (⇧⌘C)\n3. Play video\n4. Look for error messages:\n   ```\n   \"Failed to create FFmpeg demuxer: ...\"\n   \"Decoding error: ...\"\n   \"Decoder error: ...\"\n   ```\n\n**If no video frame appears**:\n- Metal rendering not running\n- Check MetalVideoView is visible in screen hierarchy\n- In PlayerScreen.swift, verify MetalVideoView is in ZStack\n\n**Check pixel buffer flow**:\n1. Add breakpoint in `decoderDidProducePixelBuffer(_:)`\n2. Verify it's being called\n3. Check pixelBuffer is non-nil\n4. Verify dimensions are > 0\n\n### Stuttering or Frame Drops\n\n**Check decoder queue**:\n- VideoDecoder runs on `DispatchQueue(label: \"com.visionuniversalplayer.decoder\")`\n- Not blocking main thread\n- Verify with Instruments → Color Blended Layers\n\n**Metal optimization**:\n- In Release build settings:\n  - Optimize for Speed: Yes (-O3)\n  - Enable LTO: Yes\n  - Metal compiler flags: -O3\n\n**Buffer size**:\n- Edit RawStreamEngine.swift:\n  ```swift\n  let buffer = StreamBuffer(chunkSize: 128 * 1024)  // Increase from 64KB\n  ```\n\n### Stats Show Wrong Resolution\n\n**Issue**: PlayerStats shows 0×0 resolution\n\n**Fix**:\n1. Verify VideoDecoder receives SPS/PPS\n   - Add logging in `parseAnnexBStream()`\n2. Check bitstream filter output\n   - FFmpegBridge.c should apply h264_mp4toannexb\n3. Verify CMVideoFormatDescriptionGetDimensions() returns valid size\n\n### Audio Not Playing\n\n**Note**: This player is VIDEO-ONLY\n\n**Currently unsupported**:\n- Audio decoding\n- Audio output\n- Sync with audio\n\n**If you need audio**:\n- Add AVAudioEngine for audio playback\n- Sync PTS between video/audio\n\n## Network Issues\n\n### \"Failed to download stream\"\n\n**Check network permissions**:\n- Info.plist must have `NSLocalNetworkUsageDescription`\n- For FTP: `NSBonjourServices` includes `_ftp._tcp`\n- For HTTP: standard (already included)\n\n**Test connectivity**:\n```bash\n# HTTP\ncurl -I http://your-server.com/video.mp4\n\n# FTP\nftp your-ftp-server.com\n\n# Check DNS\nnslookup your-server.com\n```\n\n### \"Connection timed out\"\n\n**In URLSessionConfiguration**:\n```swift\nconfig.timeoutIntervalForResource = 300  // 5 minutes\nconfig.waitsForConnectivity = true\n```\n\nChange timeout in relevant engine file if needed.\n\n### Protocol Not Working\n\n**HTTP/HTTPS**: Built-in, always works\n\n**FTP/WebDAV**: Requires FFmpeg support\n- Verify FFmpeg was compiled with these protocols:\n  ```bash\n  ffmpeg -protocols | grep -E \"ftp|webdav\"\n  ```\n- If missing: `brew reinstall ffmpeg --with-options`\n\n## Debugging Tools\n\n### Xcode Console\nWatch live logs:\n```\nProduct → Scheme → Edit Scheme → Run → Console\n```\n\n### Metal Debugger\n1.
